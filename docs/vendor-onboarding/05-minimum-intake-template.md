# 05 — Minimum Intake Template

**Purpose:** A short intake for teams who don't want to fill the full form. Enough to produce a useful first pass.

---

Copy and answer these six questions:

```
MINIMUM INTAKE

1. What do you want to build?
   (2–3 sentences.)

2. Who will use it?

3. What should the AI do?
   (Retrieve, decide, generate, approve, evaluate, automate…)

4. What data does it need?

5. What stack do you prefer (or already use)?

6. What output do you want from us?
   (e.g. file list, context pack, architecture blueprint, agent handoff pack)
```

## Notes

- This is enough for a **Fast Path** result (top repos, key files, rough architecture shape).
- For a **Deep Blueprint** (full architecture, roadmap, risk, confidence, agent handoff), the full form in `04` gives a meaningfully better outcome.
- If your minimum intake scores low on the Context Quality Score (`06`), we'll ask 2–3 targeted follow-ups before building anything — we don't fill gaps with assumptions.
